# Deployment Process — Standard Practice

## Workflow
1. **Code locally** → Write changes to local files in workspace
2. **Push to GitHub** → Commit and push (NO deploy yet)
3. **Preview/Draft** → Share link or code draft in chat for review
4. **Manual Deploy** → Only deploy after explicit approval from Arron/Taylor

## Rules
- ❌ No automatic deploys
- ❌ No live site changes without approval
- ✅ Changes must be reviewed in chat first
- ✅ Manual deploy only after approval

## Preview Options
- GitHub Pages link (auto-deploys from main branch)
- Direct code review in chat
- Test file URL

## Why This Matters
- Saves build minutes on hosting platforms
- Prevents credit exhaustion
- Allows quality control before launch
- Gives team time to review
